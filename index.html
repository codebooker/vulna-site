One appliance. Every site. Every finding under your control.

Vulna is a self-hosted security assessment and vulnerability management platform. Scan networks directly from the appliance, run scanners on a remote VulnaScout, or reach a constrained site through VulnaRelay. No vendor cloud or license key is required.

~ cd vulna && cp .env.example .env

From first discovery to verified remediation.

Vulna coordinates proven scanners, enforces approved scope, builds a durable asset inventory, explains risk, and gives teams a clear path from finding to report to verified fix.

01

Assessment from discovery to web

Drive nmap, Nuclei, testssl.sh, and OWASP ZAP through signed workflows. Run service discovery, vulnerability and TLS checks, gated web assessment, and explicitly approved controlled pentest sessions.

02

Three ways to reach a network

Use the appliance's bundled local Scout, install a remote VulnaScout where scanners should run at the edge, or use a scanner-free VulnaRelay to tunnel approved traffic from central scanners.

03

Inventory with real context

Track assets, services, observations, and material changes. Add read-only SSH or WinRM software inventory, passive source connectors, tags, dynamic groups, ownership, and reversible reconciliation.

04

Explainable risk, not a mystery score

Enrich findings with NVD, CISA KEV, EPSS, and advisories. Versioned risk profiles retain normalized inputs, factor contributions, decisions, acceptance expiries, and score history.

05

From finding to verified fix

Build remediation units, assign owners, apply policy-driven SLAs, synchronize Jira, GitHub, GitLab, GLPI, or a generic ticket system, and close the loop with a targeted verification scan.

06

Reports for every audience

Generate executive, technical, pentest, or full-spectrum PDFs plus permission-scoped CSV and JSON exports. Reuse templates, filter by site or asset group, and schedule recurring report definitions.

07

Identity and accountability

Use granular RBAC, scoped grants, service accounts, MFA, WebAuthn, OIDC or SAML SSO, and SCIM provisioning. Step-up checks and the audit log protect and attribute sensitive actions.

08

Operate it on your terms

Back up, restore, update, roll back, monitor, and troubleshoot the appliance yourself. Offline workflows are supported, integrations are opt-in, and optional aggregate telemetry is off by default.

Start with one appliance. Extend only when you need to.

The appliance includes a local Scout, so it can assess any approved network it can already reach. Add a remote endpoint only for another location, VLAN, DMZ, or routing boundary.

DeploymentWhere scanners runEndpoint needsChoose it when
Central appliancebundled local Scout On the appliance 2 CPU · 4 GB RAM · 20 GB The appliance can route to the network. No separate endpoint is required.
VulnaScoutrecommended remote option On the remote Scout Linux amd64/arm64 · Nmap A site can run scanners locally and enforce signed scope at the edge.
VulnaRelayadvanced, opt-in On the appliance, through the tunnel Linux · root · WireGuard · IPv4 The remote host should be scanner-free and provide only a kill-switched, scope-controlled egress path.

Native appliance, Scout, and Relay services currently require Linux on amd64 or arm64. Windows and macOS are supported as browser clients and assessment targets, but not yet as native endpoint-service hosts. Windows software inventory runs remotely through WinRM from an eligible Linux Scout.

Start the central appliance

Review .env and set its required secrets before starting the stack.

$ cd vulna
$ cp .env.example .env

Open source, the whole stack.

Vulna is AGPL-3.0 and developed in the open on GitHub. There is no vendor control plane, paid tier, or open-core feature gate. Findings, evidence, credentials, and keys stay on your infrastructure unless you deliberately configure an external identity, notification, ticketing, or inventory integration.

Bug reports, pull requests, and "it broke on my weird hardware" issues are all welcome. Good first issues are labeled.
